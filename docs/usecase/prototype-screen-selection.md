# ユースケース: 画面を選択してプロトタイプ（プレビュー）を生成する

## 概要

| 項目 | 内容 |
| --- | --- |
| ユースケース名 | 画面を選択してプロトタイプ（プレビュー）を生成する |
| アクター | ビルダー利用者（ログイン済みユーザー） |
| 目的 | ナビゲーション（画面）の中から作りたい画面だけを選んで、クリック可能な HTML プロトタイプを生成する。出力量を抑えて生成の途中切れ・タイムアウトを避ける。 |
| 対象画面 | `/studio/[id]/prototype`（プロトタイプ画面・右ペインのアクションツールバー＋プレビュー） |
| エンジン | AWS（Claude 等で自己完結 HTML を生成。`engine="aws"`、`mode="create"`） |

## 事前条件

- ユーザーがログイン済みで、対象プロジェクトの所有者である。
- 分析パイプラインでナビゲーション（メインナビ＝画面一覧）が生成済みである。
  - ナビが空の場合は画面選択UIは表示されず、全体（他のコンテキスト）から生成する。
- モデル設定（基準モデル／工程別モデル）が読み込まれている。

## 事後条件

- 成功時: 選択した画面で構成されたプロトタイプ HTML が DB（`prototypes`）に保存され、右ペインにプレビュー表示される。「生成された画面」一覧に生成済み画面が並ぶ。
- 失敗時: HTML は保存されず、エラー／途中切れの理由が画面に表示される。プロジェクトの他データは変化しない。

## 基本フロー（メインシナリオ）

1. ユーザーがプロトタイプ画面を開く。
2. システムはプロジェクトのナビゲーション項目を読み込み、**全画面を選択済み**の状態で「生成する画面」チップ群を表示する（`選択数/総数`）。
3. ユーザーが、作りたい画面のチップをトグルして対象を絞り込む（`全選択` / `全解除` も可）。
4. ユーザーが「プレビューを生成」を押す。
5. システムは生成ジョブを起動し（`POST /api/jobs`、`kind="prototype"`、`mode="create"`）、即座に `jobId` を返す。
6. 生成本体はサーバ側 `after()` で実行され、ナビゲーションを**選択画面に絞った**コンテキストと「指定画面だけを過不足なく実装せよ」という指示で HTML をストリーミング生成する。
7. システムは進捗をポーリング表示する（受信文字数、`@screen` マーカーから抽出した「生成できた画面」のライブ一覧）。
8. 生成が完了すると、システムは HTML を保存し、ジョブを完了にする。
9. クライアントはプレビュー（`iframe srcDoc`）に HTML を表示し、「生成された画面」一覧を表示する。
10. ユーザーは「生成された画面」チップをクリックして、プレビュー内の該当画面へ移動できる。

## 代替フロー

- **A1. 全画面を選択（既定）**
  - ステップ3で絞り込まず全選択のままにすると、`navigation` は絞られず、`selectedScreens` は送らない（＝従来どおり全画面を生成）。
- **A2. 既存プロトタイプの再生成**
  - すでに HTML がある場合、ボタンは「プレビュー再生成」となる。選択内容に従って作り直す。
- **A3. チャット相談からの再生成**
  - 統合チャット（AI相談）からの再生成（`override.nav` あり）では画面選択フィルタは適用されず、チャットが指定したナビで生成する。
- **A4. 生成中の画面離脱・リロード**
  - 生成はサーバ側で継続するため、画面遷移・リロード・タブ閉じでも止まらない。再訪時、進行中ジョブを拾い直して進捗・最終結果を反映する。

## 例外フロー

- **E1. 画面を1つも選んでいない**
  - ナビがあるのに選択が0件のとき、「プレビューを生成」は無効化され、「※1つ以上選択してください」を表示する。
- **E2. 出力上限による途中切れ**
  - 生成が出力トークン上限（プロトタイプは約16,000トークン）に達すると `finishReason="length"` を検知し、部分 HTML を保存したうえで「途中で切れた可能性」を警告表示する。→ 画面をさらに絞って分割生成するよう促す。
- **E3. 生成タイムアウト（時間切れ）**
  - 生成がサーバ関数の実行上限（300秒, Hobby プラン）を超えると打ち切られ、ジョブは「生成が時間内に完了しませんでした…画面を絞って分割を」というエラーになる。→ 画面選択での分割で回避する。
- **E4. 生成失敗（モデル/通信エラー等）**
  - ジョブが失敗としてエラーメッセージを保持し、画面に表示する。HTML は保存されない。

## ビジネスルール

- **BR1. 絞り込み判定**: `0 < 選択数 < 総数` のときだけ「部分生成」とみなし、`navigation` を選択画面（＋選択した親に属する子画面）に絞り、`selectedScreens` をプロンプトに渡す。全選択時・チャット override 時は絞らない。
- **BR2. 既定は全選択**: 画面ロード時に全画面を選択状態にする（取りこぼし防止）。
- **BR3. 親子の扱い**: 選択画面の子（`parent` が選択ラベル）も生成対象に含める。
- **BR4. 出力上限**: プロトタイプ生成の出力上限は 300 秒以内に完結する量（約16kトークン）に固定。これを超える規模は E2/E3 として扱い、分割を促す。
- **BR5. 途中切れの可視化**: 部分生成は無言で保存せず、必ず警告とともに提示する。
